## 目的

エージェント駆動開発が隆盛しており、エージェントとの会話に中毒になってしまい、しばしば寝るのを忘れてしまったり、本当にやるべきタスクを見失ってサイドプロジェクトばかりをやってしまったりする。それを防ぐために、より意図的に時間の使い方を定められるようなターミナルを作りたい。

さらに、複数のエージェントと並列に対話しながら開発することはなかなか疲れる。この問題も解決したい。

自分では「脳に優しいターミナル」と呼んでいる。

具体的には、あるエージェントにバックグラウンドタスクを依頼したら、次の時間までそれを完全に忘れられるようなシステムを作りたい。今はなんだかんだ進捗を見てしまったり、エージェントの完了通知を気にして別タスクに集中できなかったりしている。

そのため、人間にミーティングでタスクを依頼するのと同じように、AI と **Sync session**（テキストで行うミーティングのようなもの）を一定時間行い、そこで要件定義・次回 Sync session のセットを行い、次にその時間が来るまでは別タスクを行う、という感じにしたい。

## 作りたいもの

- 全体の流れ
    - 最初に要件定義・Permission・次回予定を設定する Sync session（30 min くらい）
        - 次の Sync session までに必要な Permission を話し合って事前に設定しておくことで、権限によって中断されることがないようにする
    - **Self-running**（バックグラウンドで自走する時間）
        - このとき caffeinate などを使って中断されないようにする or 中断されても再開されるようにする
        - 重いタスクをやるときは CPU 利用率が少ない時間帯を狙うとかする
        - この間は進捗を見られず、完全にバックグラウンドプロセスになる
            - 「どうしても今必要だから進捗を見たい！」ボタンをつけるのもいいのかな、やってみて考える
            - エージェントが起動している（自走中 or 完了していればよい；何かしらの原因でエラーになったり止まったりしていなければ OK）ことは、MenuBarExtra などで分かるようにしたい
    - 次の Sync session の時間になったら、進捗共有、ここまでの稼働率レビュー、Permission を含めた harness の見直しについて考える
        - 稼働率レビュー：例えば前の Sync session から今の Sync session までが 24 時間あっても、実際にエージェントが動いていたのは 1 時間くらいしかないかもしれない。AI に自走させる時間を極力長くしていくためには、このメトリクスを取って定期的に振り返るとよい
- アプリの全体像
    - 基本的にはこのリポジトリ（cmux のフォーク）に機能を追加していくイメージ
    - AI と設定している Sync session の時間をカレンダービューで見られる
    - バックグラウンドで AI に任せるタスクは **Async workspace**（＝プロジェクト）と呼ぶ
        - 新しい Async workspace を作成すると、初回 Sync session ができる
    - Async workspace の一覧をどこかで見られる必要はありそうだがまだあまり考えてない
